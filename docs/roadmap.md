# ConnFox 实施路线

## P0：现在就能做的

因为 Flutter 还没装完，当前先做这些不阻塞的部分：

- 建项目骨架
- 做桌面工作台原型
- 设计窗口 / Tab / 连接状态模型
- 规划后续驱动接口
- 先把 README 和实施文档补齐

## P1：Flutter 安装完成后的第一轮

优先级从高到低：

1. 让 macOS 工程可运行
2. 做连接配置表单
3. 接 MySQL 驱动
4. 做 SQL 执行和结果集展示
5. 做 Schema 浏览和懒加载

这一阶段的目标不是“功能很多”，而是“主链路完整”：

- 新建连接
- 保存连接
- 打开连接
- 开多个 Tab
- 执行 SQL
- 看结果
- 创建表
- 看表关系图

## P2：对齐专业工具的常用能力

- 查询历史
- 收藏 / Snippets
- 结果导出
- 表结构编辑
- 表数据编辑
- 关系图导出
- 只读模式
- 危险 SQL 二次确认
- 连接分组和标签
- 自动重连
- SSH / SSL

## P3：桌面强化

- 真实多窗口
- 快捷键体系
- 命令面板
- macOS 标题栏和菜单集成
- 窗口恢复
- 更大结果集下的流畅滚动

## P4：扩数据库类型

前提是先把这些抽象稳定下来：

- 驱动接口
- 连接参数模型
- 元数据模型
- 查询结果模型
- capability 开关

然后按数据库一个一个接：

1. PostgreSQL
2. SQLite
3. SQL Server

## 你现在最合适的开发顺序

如果你想先尽快看到成型效果，建议照这个顺序走：

1. 先跑起来 macOS 壳层
2. 先接 MySQL 连接
3. 再做结果表格和 Schema
4. 再做建表、数据编辑、关系图
5. 再做历史、收藏、导出
6. 最后补多窗口和更多数据库
